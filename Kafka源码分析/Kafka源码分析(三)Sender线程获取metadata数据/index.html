Kafka源码分析(三)Sender线程获取metadata数据

一、前言

  • 前面KafkaProducer 类里面开启sender线程。

二、Sender线程执行流程图


sequenceDiagram
autonumber
KafkaProducer ->> Sender: ioThread.start()
Sender->>Sender: run()多线程方法
loop while (running)[死循环]
    Sender->>Sender: 获取当前集群metadata信息 metadata.fetch();
    Sender->>Sender: 获取已准备好发送数据的分区请求
    Sender->>Sender: 创建批次请求this.accumulator.drain
    Sender->>Sender: 保证消息顺序，guaranteeMessageOrder：false
    Sender->>Sender: 准备待发送请求的数据，sendProduceRequests(batches, now);
    Sender->>NetworkClient: this.client.poll(pollTimeout, now);
    NetworkClient->> NetworkClient: 封装要拉取元数据的请求metadataUpdater.maybeUpdate(now);
    NetworkClient->> Selector: 阻塞等待，直到有可用连接或超时select(timeout);
end

三、源代码分析

  • Sender线程的run方法被执行
  • 拉取元数据的请求，在第一次执行的时候，不会返回任何的数据
  • 拉取元数据，给定超时间返回可用的通道或超时
